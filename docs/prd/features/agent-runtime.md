# Agent Runtime

## Overview

에이전트 spec을 읽어서 실행하는 코어 엔진. 스텝을 순서대로 실행하고, 모드(AI/AI+Approve/Manual)에 따라 처리를 분기하고, 컨텍스트를 주입하고, 각 스텝의 결과를 수집한다. Zova 파이프라인의 실행 계층.

---

## Requirements

- REQ-001: agent-spec YAML 파일을 로드하고 validation한 뒤 실행
- REQ-002: 스텝을 정의된 순서대로 순차 실행
- REQ-003: 스텝 모드별 처리:
  - `ai` — LLM API 호출 후 결과를 자동으로 다음 스텝에 전달
  - `ai_approve` — LLM API 호출 후 결과를 사용자에게 표시. 승인(approve) 시 다음으로, 거절(reject) 시 재생성 또는 수동 편집 선택
  - `manual` — 사용자 입력 대기. 입력 완료 시 다음 스텝으로
- REQ-004: 프롬프트 템플릿의 변수(`{{steps.*.output}}`, `{{article.*}}`)를 실행 시점에 실제 값으로 치환
- REQ-005: 글로벌 컨텍스트(에이전트 레벨)와 스텝 컨텍스트를 병합하여 프롬프트에 주입
- REQ-006: 각 스텝의 실행 결과(output)를 저장하고, 이후 스텝에서 참조 가능하게 유지
- REQ-007: 에이전트 실행 완료 시 전체 결과(모든 스텝의 output)를 반환
- REQ-008: 스텝 실행 중 LLM API 에러 발생 시 재시도 (최대 3회). 재시도 실패 시 해당 스텝에서 정지하고 사용자에게 알림
- REQ-009: 실행 로그 — 각 스텝의 시작/종료 시간, 모드, 토큰 사용량 기록

---

## User Journeys

### 에이전트 실행 (전체 AI 모드)
1. 사용자가 에이전트 실행 명령 (CLI)
2. 런타임이 spec 파일 로드 + validation
3. 스텝 1부터 순차 실행. 각 스텝에서 프롬프트 변수 치환 → LLM 호출 → 결과 저장
4. 모든 스텝 완료 후 최종 결과 출력
5. 실행 로그 표시 (스텝별 소요시간, 토큰 사용량)

### AI+Approve 모드 스텝 처리
1. 런타임이 LLM을 호출하여 결과 생성
2. 결과를 사용자에게 표시
3. 사용자 선택: approve → 다음 스텝 / reject → 재생성 or 직접 편집
4. 직접 편집 선택 시 → 에디터에서 수정 → 수정본을 결과로 사용

### Manual 모드 스텝 처리
1. 런타임이 해당 스텝에서 일시 정지
2. 이전 스텝들의 결과를 컨텍스트로 표시
3. 사용자가 내용을 직접 입력
4. 입력 완료 시 다음 스텝으로 진행

### 에러 발생
1. LLM API 호출 실패
2. 자동 재시도 (최대 3회, 지수 백오프)
3. 재시도 실패 시 해당 스텝에서 정지
4. 사용자에게 에러 내용 표시 + 선택지: 재시도 / Manual로 전환 / 중단

---

## Edge Cases

- 첫 스텝이 manual인 경우 → 바로 사용자 입력 대기 (이전 컨텍스트 없음)
- ai_approve에서 연속 reject → 무제한 재생성 허용. 사용자가 직접 편집 또는 중단 선택
- 스텝 중간에 실행 중단 후 재개 → MVP에서는 미지원. 처음부터 재실행
- LLM 응답이 기대 포맷에 맞지 않음 → 재시도. 3회 실패 시 raw output 표시하고 사용자에게 위임
