# Zova — Product Brief

**Date:** 2026-04-11 | **Status:** Discovery
**Tagline:** AI 에이전트들이 콘텐츠를 만드는 파이프라인. 각 스텝의 자동화 수준은 사용자가 정한다.

---

## Problem

**What:** 다채널 콘텐츠 운영자는 아이디어 발굴부터 채널별 발행까지 전 과정을 반복한다. 채널마다 미디어 특성이 달라서 — 유튜브는 스크립트·씬·썸네일, 블로그는 SEO·본문·메타, X는 훅·쓰레드 분할 등 고유한 제작 과정이 필요하다. AI 도구를 써도 채널마다 수동 반복이고, 어디까지 자동화하고 어디서 직접 개입할지 선택할 수 없다.

**Who and when:** 다채널 콘텐츠를 운영하는 크리에이터, 마케터, 소규모 미디어 팀. 도메인 불문. 주 3회 이상 발행 시 병목이 심해진다.

**Why now:** LLM 품질이 실사용 수준에 도달했지만, 콘텐츠 제작의 전체 흐름을 에이전트로 구조화하고 자동화 수준을 조절할 수 있는 도구가 없다.

---

## Direction

Zova는 콘텐츠 제작 전체를 에이전트 파이프라인으로 구성한다.

**에이전트**
리서치, 채널 변환 구분 없이 모든 기능 단위가 에이전트. 각 에이전트는 서브 파이프라인(여러 스텝)을 가진다.
- 리서치 에이전트: 트렌드 수집 → 아이디어 발굴 → 소스 리서치 → 원본 초안
- 유튜브 에이전트: 스크립트 → 씬 분할 → 보이스오버 → 썸네일 프롬프트 → 설명문
- 블로그 에이전트: SEO 키워드 → 본문 포맷팅 → 메타 디스크립션 → 대표 이미지
- X 에이전트: 훅 트윗 → 쓰레드 분할 → CTA
- 사용자가 에이전트를 추가하거나 기존 에이전트의 스텝을 수정 가능

**스텝 모드**
모든 스텝은 세 가지 모드 중 하나로 설정:
- **AI** — 에이전트가 자동 실행, 다음 스텝으로 바로 진행
- **AI + Approve** — 에이전트가 실행하되 사용자 승인 후 다음으로
- **Manual** — 사용자가 직접 입력/편집

같은 파이프라인이라도 스텝별 모드 조합으로 완전 자동 ~ 거의 수동까지 조절 가능.

**원본 아티클**
에이전트 간 데이터를 주고받는 구조화된 공유 포맷. title, hook, tldr, sections, keyPoints, dataPoints, sources 등. 리서치 에이전트가 만들고, 채널 에이전트들이 소비한다.

**에이전트 런타임**
에이전트를 실행하는 코어 엔진. 스텝 실행, 모드 처리, 컨텍스트 주입, 결과 수집.

**Core bet:** 콘텐츠 제작의 모든 단계(아이디어 → 리서치 → 원본 → 채널별 제작)를 하나의 에이전트 모델로 통일하고, 각 스텝의 자동화 수준을 사용자가 직접 조절하게 하면 — 도메인과 워크스타일에 상관없이 동작한다.

---

## Success Signal

사용자가 자기 도메인에 맞게 에이전트 프롬프트를 설정하고, 스텝별 AI/Manual 모드를 조합해서 자기만의 콘텐츠 파이프라인을 만든다. 원본 아티클 하나에서 여러 채널 콘텐츠가 나오고, 반복 사용하면서 파이프라인을 튜닝해간다.
